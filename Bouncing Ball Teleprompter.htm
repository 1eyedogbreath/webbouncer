Phase 1: JSON Configuration System

Core JSON Architecture

JSON Example:

{
  "title": "The Rainbow Connection",
  "content_type": "song",
  "text_file": "rainbow_connection.txt",
  "bounces_per_line": 4,
  "bounce_duration": 1.625,
  "beat_pattern": [1, 4, 7, 10]
}

Content Type Support

  • Songs: Chords + lyrics + beats, complex parsing
  • Poems: Text + beats only, syllable emphasis (iambic pentameter)
  • Speeches: Simple text + timing markers, word-level highlighting
  • Podcasts: Timestamp-based highlighting
  • Plays: Character names + dialogue, scene markers

Phase 2: Python UI Enhancement

Implementation Order (One at a time, with testing)

  1. File Prompt → JSON menu system

  3. Visual Speed Display → Show current speed on screen
  4. Play/Pause Button → pygame button class
  5. Restart Button → pygame button

Technical Architecture Goals

Configuration Management

Python Classes:

class PerformanceConfig:
    def load_from_json(self, filename)
    def save_user_preferences(self)
    def get_content_parser(self)  # Returns song/poem/speech parser
    def get_timing_engine(self)  # Returns appropriate timing system

Modular Content Parsers

class SongParser:     # Handles chords + lyrics + beats
class PoemParser:     # Handles text + syllable patterns
class SpeechParser:   # Handles text + timing markers
class PlayParser:     # Handles characters + dialogue
